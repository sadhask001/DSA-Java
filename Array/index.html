Array Basics

What is an Array?

An array is a data structure used to store a collection of elements of the same data type in contiguous memory locations. Instead of declaring separate variables for each value, an array holds multiple values under a single name using index-based tracking offsets starting at index 0.

Features: Homogeneous Data, Fixed Size, Continuous Memory, Random Access ($O(1)$)

Advantages

  • Instant elements lookup using index references.
  • Excellent cache locality due to continuous blocks alignment.
  • Memory efficient for storing sequential entries.

Disadvantages

  • Fixed static sizing allocations limits; cannot change size at runtime.
  • Shifting overhead cost ($O(n)$) incurred during structural insertions/deletions.
🌍 Real-Life Analogy: Think of an array like a row of numbered lockers in a school hallway. Each locker is the exact same size, sits sequentially next to the other, and can be instantly accessed if you know its specific locker number (index).

Memory Allocation Concept & Formula

Because arrays occupy structured continuous hardware segments, the system does not need to scan through previous elements to access a targeted slot. It computes the physical space offset instantly using the standard **Random Access Addressing Formula**:

Address = Base Address + (Data Type Size × Index Position)

Formula Variables Breakdown:

  • Base Address: The initial physical RAM starting pointer where the array's index `[0]` sits.
  • Data Type Size: The byte footprint consumed by a single element data cell (e.g., `Integer = 4 Bytes`, `Double = 8 Bytes`, `Byte = 1 Byte`).
  • Index Position: The absolute zero-indexed offset parameter targeting the value slot.

Example Mathematical Trace: Suppose an integer array starts at Base Address `1000`. To access index position `[3]`, the memory engine maps the configuration instantly: $\text{Address} = 1000 + (4 \times 3) = 1012$.

Memory Layout: Stack vs Heap

In Java, arrays are treated as objects. Therefore, memory distribution is split into two areas:

  • Stack Memory: Stores the array reference variable pointer name.
  • Heap Memory: Allocates the actual continuous memory blocks containing array element boxes and default values.
// Memory Mapping Lifecycle Trace
int[] arr;          // Stack variable pointer reference created
arr = new int[4];   // Heap blocks memory allocated sequentially: [0, 0, 0, 0]

Basic Operations

1. Create Array

📖 Definition: Allocates a fixed-size contiguous memory block segment for elements of a declared data type.

🎯 Purpose: Reserves room in system heap memory before setting explicit elements entries values.

💻 Java Code Implementation

int[] arr = new int[5];

📥 Input / 📤 Output Data

Input: Size = 5
Output: [0, 0, 0, 0, 0] (Initialized with defaults)
📝 Step-by-Step Explanation: Java checks the declared variable size parameter, finds a continuous open chunk of 20 bytes (5 elements $\times$ 4 bytes for Integer) in the heap, zeroes them out, and points the stack reference pointer directly to its base memory address.
Time: O(1) Space: O(N)

📖 Definition: Evaluates elements keys individually step-by-step from index 0 until the matching criteria filter key is hit.

    if(arr[i] == target) return i; // Found

8. Delete Element

📖 Definition: Extracts value parameters datasets, shifting all remaining adjacent trailing frames one slot left to preserve compaction sequence parameters configurations.

💻 Java Code Implementation

// Deleting value at index 2 by shifting left
for(int i = 2; i < arr.length - 1; i++) {
    arr[i] = arr[i + 1];
}

Utility Operations

Inversion, Replication, and Analytical Searching API Blocks

Utility macros manipulate array properties. These workflows can reverse arrays in place, build isolated deep clones inside the heap, or check data equity profiles.

🔄 Reverse Array

Two-pointer swap method loops inward to mirror array boundaries entries.

In: [10, 20, 30, 40] → Out: [40, 30, 20, 10]
Time: O(N) Space: O(1)

🔍 Binary Search Engine

Splits sorted array ranges in half repeatedly to locate matching query targets in logarithmic time profiles.

Sorted In: [10, 20, 30, 40] → Search: 30 → Out: Index 2
Time: O(log N) Space: O(1)

Mathematical Operations

Statistical and Frequency Profile Analytics

Linear $O(n)$ scanning algorithms evaluate collection aggregates and frequency maps by checking each block cell sequence item.

  • Sum & Average: Accales value counters down through loops.
  • Maximum / Minimum boundary filters: Tracks the highest or lowest numeric value seen while traversing.
  • Second Largest element lookup: Tracks the two highest variables values simultaneously in a single loop scan pass.
  • Frequencies profiling maps: Counts duplicate occurrences across target variable keys.

Array Rearrangement

Rotations, Shift Compactions, and Pointer Swaps

🔄 Rotate by K Shifts

Shifts element indices out of bounds and wraps them around. This is usually optimized using a 3-step block inversion strategy: `reverse(0, k-1)`, `reverse(k, n-1)`, then `reverse(0, n-1)`.

Time: O(N) Space: O(1)

🧹 Move Zeroes to End

Uses a read/write pointer model. It drops active non-zero elements into early sequence cells before zero-filling all trailing memory block cells.

Time: O(N) Space: O(1)
