📖 Introduction

Asymptotic Notations are mathematical tools used to measure the efficiency of an algorithm as the input size (n) increases.

Instead of measuring execution time in seconds, they measure the growth rate of an algorithm.

💡 Why don't we calculate execution time?

  • Different computers execute programs at different speeds.
  • Programming languages have different performance.
  • Compilers optimize code differently.
  • Input size changes execution time.

Therefore, we use Asymptotic Analysis to compare algorithms independent of hardware.

Asymptotic Analysis

Asymptotic Analysis studies how an algorithm behaves when the input size (n) becomes very large.

It ignores constants and lower-order terms and focuses only on the dominant growth. 


Example

T(n)=5n²+4n+10

For very large n

T(n)=O(n²)

🔴 Big O Notation ( O )

Big O represents the Upper Bound of an algorithm.

It tells the maximum amount of work an algorithm may perform.

Represents

Worst Case

Mathematical Definition


T(n) ≤ C × f(n)

for n ≥ n₀

where

T(n) = Actual Running Time

f(n) = Complexity Function

C = Positive Constant

n₀ = Starting Point

🟢 Big Omega Notation ( Ω )

Big Omega (Ω) represents the Lower Bound of an algorithm.

It tells us the minimum amount of work an algorithm performs.

Represents

Best Case

Mathematical Definition


T(n) ≥ C × f(n)

for n ≥ n₀

where

T(n) = Actual Running Time

f(n) = Complexity Function

C = Positive Constant

n₀ = Starting Point

Java Example


int linearSearch(int arr[], int key){

    for(int i=0;i<arr.length;i++){

        if(arr[i]==key)
            return i;

    }

    return -1;

}

Found immediately

Ω(1)
Space Complexity 

O(1)

Why is it Ω(1)?

The element is found during the first comparison.

Only one comparison is required.

Minimum operations remain constant. 


Ω(1)

🔵 Big Theta Notation ( Θ )

Big Theta (Θ) represents the Tight Bound.

It tells us that the algorithm grows at approximately the same rate for both the upper and lower bounds.

Represents

Exact Growth

Mathematical Definition



C₁ × f(n)

≤ T(n)

≤ C₂ × f(n)

for n ≥ n₀

Java Example



void mergeSort(int arr[],int l,int r){

    if(l<r){

        int mid=(l+r)/2;

        mergeSort(arr,l,mid);

        mergeSort(arr,mid+1,r);

        merge(arr,l,mid,r);

    }

}

Step-by-Step Calculation

Merge Sort divides the array into two equal halves.

Number of divisions 



log₂ n

At every level, all elements are merged. 



n

Total Work 



n × log n
Time Complexity 


Θ(n log n)
Space Complexity 


O(n)

Why is it Θ(n log n)?

Case Complexity
Best Θ(n log n)
Average Θ(n log n)
Worst Θ(n log n)

Regardless of whether the array is

  • Already Sorted
  • Reverse Sorted
  • Randomly Ordered

Merge Sort always performs the same sequence of

  • Divide
  • Merge

Hence, 



Θ(n log n)

💡 Rule of Thumb

For large datasets, prefer algorithms having O(1), O(log n), O(n), or O(n log n) whenever possible.

🎯 Interview Tips

  • Always explain both Time Complexity and Space Complexity.
  • Count the number of loops carefully.
  • Nested loops generally result in O(n²).
  • Halving the input usually results in O(log n).
  • Ignore constants while calculating complexity.
  • Focus on the highest-order term.
  • Recursive algorithms should also consider recursion depth.
  • Space Complexity includes recursion stack memory.

Common Interview Questions

  • What is the difference between Big O and Big Θ?
  • Why is Merge Sort Θ(n log n)?
  • Why is Binary Search O(log n)?
  • Why is Bubble Sort O(n²)?
  • Can an algorithm have different Best and Worst Cases?
  • Why do we ignore constants in Big O?

📚 Summary

  • Big O measures the Worst Case.
  • Big Ω measures the Best Case.
  • Big Θ measures the Exact Growth.
  • Time Complexity measures execution growth.
  • Space Complexity measures memory growth.
  • Efficient algorithms help solve large problems faster.
  • Always compare algorithms using asymptotic analysis instead of execution time.
