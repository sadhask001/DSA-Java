📖 Introduction

Asymptotic Notation helps us measure the efficiency of an algorithm as the input size (n) increases. Instead of measuring the execution time in seconds, it focuses on the growth rate of an algorithm. This makes it easier to compare two algorithms, regardless of the computer or programming language used.

🎯 Why Complexity Analysis?

⚡ Compare Algorithms

Choose the faster algorithm for large inputs.

🚀 Improve Performance

Reduce execution time and memory usage.

💼 Interview Ready

Almost every coding interview asks complexity questions.

🔴 Big O Notation

Big O Notation describes the Upper Bound of an algorithm. It tells the maximum amount of work an algorithm may perform for a given input size.

📖 Definition

Big O represents the Worst Case of an algorithm. It gives an upper limit on execution time.


Represents

  • Worst Case
  • Upper Bound
  • Maximum Running Time

🧮 Mathematical Formula

T(n) ≤ C × f(n)

Where,

  • T(n) → Running Time
  • f(n) → Growth Function
  • C → Constant
  • n₀ → Starting Point

💻 Java Example

for(int i=0;i<n;i++){
    System.out.println(i);
}

🧠 Step-by-Step Calculation

Step 1

The loop starts from 0 and ends at n-1

Step 2

The statement executes once for every iteration.

Step 3

Total Operations

n

Final Complexity

Time Complexity

O(n)

Space Complexity

O(1)

🟢 Big Ω (Big Omega)

Big Omega represents the Lower Bound of an algorithm. It tells the minimum amount of work an algorithm must perform.

📖 Definition

Big Omega represents the Best Case of an algorithm. It defines the minimum running time required.


Represents

  • Best Case
  • Lower Bound
  • Minimum Running Time

🧮 Mathematical Formula

T(n) ≥ C × f(n)

Where,

  • T(n) → Running Time
  • f(n) → Growth Function
  • C → Positive Constant
  • n₀ → Starting Input Size

💻 Java Example

int linearSearch(int arr[], int key){
    for(int i=0;i<arr.length;i++){
        if(arr[i] == key){
            return i;
        }
    }
    return -1;
}

🧠 Step-by-Step Calculation

Step 1

Suppose the element is located at index 0 (the first position).

Step 2

The first comparison becomes true immediately.

Step 3

Only 1 comparison is performed.

Final Complexity

Time Complexity

Ω(1)

Space Complexity

O(1)

📘 Another Example

while(low <= high){
    int mid = (low + high) / 2;
    if(arr[mid] == key)
        return mid;
}

If the required element is found at the very first middle position, only one comparison is required.

Ω(1)

🌍 Real-Life Example

Finding a Book

Imagine your favorite book is placed exactly in front of you. You immediately pick it without searching. Only one operation is needed.

Ω(1)

🔵 Big Θ (Big Theta)

Big Theta represents the Tight Bound of an algorithm. It provides both the Upper Bound and Lower Bound, meaning the algorithm grows at the same rate.

📖 Definition

Big Theta represents the Exact Growth Rate of an algorithm. If an algorithm has the same Upper Bound and Lower Bound, then it belongs to Θ(f(n))


Represents

  • Tight Bound
  • Exact Growth
  • Average Case (Generally)

🧮 Mathematical Formula

C₁ × f(n) ≤ T(n) ≤ C₂ × f(n)

for n ≥ n₀

💻 Java Example

void mergeSort(int arr[], int l, int r){
    if(l < r){
        int mid=(l+r)/2;
        mergeSort(arr,l,mid);
        mergeSort(arr,mid+1,r);
        merge(arr,l,mid,r);
    }
}

🧠 Step-by-Step Calculation

Step 1

Array is divided into two halves repeatedly.

Step 2

Number of levels = log n

Step 3

Every level processes n elements.

Final Calculation

n × log n

Θ(n log n)

Space Complexity → O(n)

🌍 Real-Life Example

Phone Directory

Imagine finding a name by repeatedly dividing the directory into halves. Every level performs the same amount of work. Therefore, Merge Sort grows consistently.

Θ(n log n)

🎯 Interview Questions

Frequently asked Time & Space Complexity interview questions.

Difference between Big O and Big Θ?

Big O represents the maximum growth (Worst Case).

Big Θ represents the exact growth when both upper and lower bounds are the same.

Why Binary Search is O(log n)?

Binary Search divides the search space into half every iteration.

Total levels → log₂ n

Why Merge Sort is O(n log n)?

Every recursive level processes n elements.
Number of levels = log n

Total = n log n

Why do we ignore constants?

As input size becomes very large, constants become insignificant.

Example: O(5n) → O(n)

Difference between Time and Space Complexity?

Time Complexity → Execution Time

Space Complexity → Memory Usage

📚 Summary

  • Time Complexity measures execution time.
  • Space Complexity measures memory usage.
  • Big O → Worst Case.
  • Big Ω → Best Case.
  • Big Θ → Exact Growth.
  • Always explain both Time & Space Complexity.
  • Ignore constants and lower-order terms.
