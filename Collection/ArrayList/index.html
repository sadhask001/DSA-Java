ArrayList in Java

Introduction

ArrayList is one of the most commonly used classes in the Java Collection Framework. It is a dynamic array that automatically grows or shrinks as elements are added or removed. Unlike a normal array, an ArrayList does not have a fixed size, making it flexible and easy to use.

ArrayList implements the List interface and stores elements in insertion order. It allows duplicate elements, supports null values, and provides fast random access using indexes. Internally, an ArrayList stores its elements in a resizable array. When the array becomes full, Java automatically creates a larger array and copies the existing elements into it.

🌍 Real-Life Example: Imagine a student attendance list containing variables tracks (John → David → Emma → Sophia). Students may join or leave at any time. Instead of initializing an entire new hardware segment every single time, we simply add a student, remove an entry, or execute a fast query scan. This is exactly how an ArrayList functions.

Features of ArrayList

  • Dynamic Sizing: Automatically expands and downscales its backing array footprints.
  • Order Preservation: Safely maintains absolute elements insertion sequences order.
  • Duplicates & Nulls: Permits duplicate values and logs multiple null values cleanly.
  • Fast Random Access: Direct lookup algorithms fetch array cells instantly via position integer indexes.
  • Thread Safety Notice: ArrayList is not synchronized (not thread-safe).

Internal Working & Growth

How Dynamic Resizing Works Internally

An empty instantiation statement creates a raw default structure container. When the active element volume matches the allocated array length boundaries, a resizing event updates structural capacities:

Resizing Equation Threshold: New Capacity = Old Capacity + (Old Capacity >> 1)
Old Capacity Limit: 10 Cells → Scales up 1.5x → New Capacity Boundary: 15 Cells

Java allocates an entirely fresh continuous backing array in heap matching the computed New Capacity parameters, seamlessly clones the historical indices items sequence over into it, and changes the internal object reference pointer address.

Pre-allocates an array mapping matching specified dimensions size bounds.

Instantiates an autonomous dynamic array containing elements duplicated from any alternate source collection framework container.

import java.util.*;

public class ArrayListExample {
    public static void main(String[] args) {
        ArrayList<String> fruits = new ArrayList<>();

        fruits.add("Apple");   // Adds Apple at index 0
        fruits.add("Banana");  // Adds Banana at index 1
        fruits.add("Orange");  // Adds Orange at index 2

        System.out.println("ArrayList : " + fruits);
        System.out.println("Size : " + fruits.size());
        System.out.println("Contains Banana : " + fruits.contains("Banana"));

        fruits.remove("Banana"); // Removes element and shifts trailing values left
        System.out.println("After Remove : " + fruits);

        for(String fruit : fruits){
            System.out.println(fruit);
        }
    }
}

Ideal Structural Contexts

  • Read-intensive applications demanding lightning-fast lookup operations.
  • Dynamic workflows where insertions and removals occur primarily at the end of the collection sequence.
  • Scenarios requiring exact duplication tracking and rigid insertion-order preservation mapping.

Scenarios to Avoid

Avoid using an ArrayList if your application requires frequent intermediate data insertions or removals in the middle of the structure sequence. Shifting trailing indices parameters left or right incurs performance costs ($O(n)$); use a `LinkedList` container configuration in those cases instead.

Technical Interview Prep

High-Frequency Assessment Q&A

1. What is the default initial capacity of an ArrayList container?

An ArrayList initiates empty without claiming heavy early hardware footprints. When the first element is added, Java sets the default backing internal array storage threshold capacity to **10 cells**.

2. Outline the explicit performance variance between arrays and ArrayList.

Standard arrays possess absolute static sizing locks and accept raw primitive primitives data fields directly. An ArrayList manages dynamic scaling properties automatically but works only with object references, relying on automatic wrapper type autoboxing transformations for primitive entries.

3. Explain how the capacity expands once the backing array grid is entirely full.

The system triggers an internal automatic expansion routine. It calculates a new structural size parameter bound to **1.5 times the current capacity**. It then allocates a fresh continuous memory block array object on the heap and copies existing elements into the updated indexes sequence layout blocks.
